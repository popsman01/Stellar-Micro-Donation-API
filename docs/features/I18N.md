# Multi-Language Error Messages (i18n)

Error messages are returned in the language requested via the `Accept-Language` HTTP header.

## Supported Languages

| Code | Language |
|------|----------|
| `en` | English (default) |
| `es` | Spanish |
| `fr` | French |
| `pt` | Portuguese |

## Usage

Send the standard `Accept-Language` header:

```
Accept-Language: es-MX,es;q=0.9,en;q=0.8
```

The response will include a `Content-Language` header indicating the language used, and error messages will be in the requested language.

## Fallback

Unsupported languages fall back to English. The quality factor (`q=`) in `Accept-Language` is respected — the highest-quality supported language wins.

## Security

- User input (`Accept-Language`) is **only** used to select a language code — it is never used as a translation key or interpolated into messages.
- Translation keys are internal constants defined in `src/utils/errors.js`.
- No user-supplied data appears in translated messages.

## Files

| File | Purpose |
|------|---------|
| `src/utils/i18n.js` | `parseLanguage()` and `getMessage()` utilities |
| `src/i18n/en.js` | English translations |
| `src/i18n/es.js` | Spanish translations |
| `src/i18n/fr.js` | French translations |
| `src/i18n/pt.js` | Portuguese translations |
